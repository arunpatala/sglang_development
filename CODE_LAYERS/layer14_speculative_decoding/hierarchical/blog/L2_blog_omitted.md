## §A — The Full Mathematical Proof

*Source: COMBINED.md §5.2, §5.3 — from Data Processing Club (09)*

*Omitted because:* The blog states the acceptance rule and the guarantee with clarity. The formal Case 1/2 decomposition proof requires set notation and is dense enough to derail readers who don't need it. The blog carries the result; this carries the proof.

---

### SpecJudge and SpecNormalize terminology

The naive approach — accept only when the draft token exactly matches the target's top-1 — fails even when draft and target are identical. With a 100-token vocabulary, the probability of exact match is 1%. A probabilistic rule is required.

**SpecJudge** (the acceptance decision):

For each position i, given draft probability `q_d(xᵢ)` and target probability `p_t(xᵢ)`:

```
(1) If q_d(xᵢ) ≤ p_t(xᵢ): Accept always.

(2) Otherwise: Accept with probability p_t(xᵢ) / q_d(xᵢ).
    Sample r ~ Uniform(0,1); accept if r < p_t / q_d.
```

**SpecNormalize** (the correction on rejection):

When a token is rejected, sample a replacement from:

```
p_resid(x) = normalize( max(0, p_t(x) - q_d(x)) )
```

---

### The decomposition proof

Define D = {x | q_d(x) > p_t(x)} — tokens where the draft overestimates.

**Case 1: x ∈ D** (draft overestimates)
- Generated by draft with probability q_d(x)
- Accepted via SpecJudge with probability p_t(x) / q_d(x)
- Net: q_d(x) × (p_t(x) / q_d(x)) = **p_t(x)** ✓

**Case 2: x ∉ D** (target assigns equal or higher probability)
- Accepted whenever generated by draft: probability q_d(x)
- When a token in D is rejected (rejection mass C = Σ_D (1 - p_t/q_d) dq_d), sample x from p_resid
- Net additional: C × (p_t(x) - q_d(x)) / C = p_t(x) - q_d(x)
- Total: q_d(x) + (p_t(x) - q_d(x)) = **p_t(x)** ✓

In both cases, the token is emitted with exactly the target's probability.

**Key property:** Standard rejection sampling may reject and re-sample repeatedly. SpecJudge guarantees resolution in **at most one round** — accept via (1), accept via (2), or reject and sample once from SpecNormalize. No re-sampling loop.

## §C — LayerSkip Full Treatment

*Source: COMBINED.md §13 — from HuggingFace LayerSkip blog (05)*

*Omitted because:* The blog covers self-speculative decoding with enough detail for awareness and contrast. The training recipe, three-cache architecture, and KVQ mechanics are implementation depth that belongs here.*

---

### The training recipe

LayerSkip requires two modifications during training:

1. **Layer dropout:** Progressively higher dropout rates for deeper layers. Trains the model to be less reliant on its later layers, improving generalization and enabling early exits.

2. **Early exit loss:** Total loss = sum of normalized losses from every intermediate exit layer:
   ```
   L_total = Σ L_exit_i  (normalized per layer)
   ```
   Forces the LM head to learn to interpret outputs from any layer — not just the final one.

Available checkpoints:
```
facebook/layerskip-llama2-7B
facebook/layerskip-llama2-13B
facebook/layerskip-llama2-70B
facebook/layerskip-llama3-8B
facebook/layerskip-llama3.2-1B
```

---

### Three caches shared between draft and verify phases

| Cache type | What it stores | Benefit |
|------------|---------------|---------|
| Shared weights | Same layers 0…E used for both phases | No weight duplication |
| Shared KV cache | Key-value pairs from layers 0…E cached in draft phase | Verification skips recomputing early layers |
| Exit query cache (KVQ) | Query vector from layer E saved | Verification resumes seamlessly from draft handoff |

The KVQ cache is the key engineering insight: verification picks up exactly where drafting left off, paying only for the layers that weren't already computed.

---

### Inference code

```python
from transformers import AutoTokenizer, AutoModelForCausalLM

early_exit_layer = 4
checkpoint = "facebook/layerskip-llama2-7B"

tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForCausalLM.from_pretrained(checkpoint).to("cuda")

inputs = tokenizer("Alice and Bob", return_tensors="pt").to("cuda")
outputs = model.generate(**inputs, assistant_early_exit=early_exit_layer)
```

---

### The early exit tradeoff

- **Exit earlier** → draft tokens faster but less accurate → lower acceptance rate.
- **Exit later** → draft tokens more accurate but slower → closer to full model speed.

The optimal early exit layer is a hyperparameter that depends on model size and task domain. No single value works across all settings.

### Related self-speculative approaches

- **Draft & Verify:** skips pre-determined attention and feed-forward layers using Bayesian optimization. Speedup up to 1.99×. Single model deployment.
- **MagicDec:** uses a subset of the KV cache as the early exit signal. Useful for long-context workloads.
- **Jacobi / Lookahead Decoding:** random or n-gram guess tokens as draft. No separate model, no training.

## §E — Extensions: DistillSpec, Online SD, Tree Attention, Medusa

*Source: COMBINED.md §14 — from Data Processing Club (09)*

*Omitted because:* These are L3/L4 depth — each warrants its own article. The blog mentions the variant landscape; this is the full landscape extension.*

---

### DistillSpec — align draft to target via knowledge distillation

The subtle trap: higher token-level accuracy of the draft model does **not** always improve speculative decoding. If the draft is *more confident* than the target about a token the target is uncertain about, that token will be rejected — even if it's "correct" from a ground-truth perspective.

**DistillSpec fix:** rather than training the draft against ground-truth labels, distill from target → draft to minimize KL(p_t || q_d). The draft learns to approximate the target's *distribution*, not the training labels.

Result: **10–45% additional speedup** on top of baseline speculative decoding.

---

### Online Speculative Decoding — fine-tune the draft on the fly

After each verification step, store mismatches (tokens where target rejected the draft) in a training buffer. Continuously fine-tune the draft on those mismatches. Adapts to shifts in production query distribution without pre-collecting training data. Especially effective for domain-specific deployment where production prompts differ from training data.

---

### Tree attention — verify multiple branches simultaneously

Instead of one draft sequence, generate a **tree** of candidate continuations. Tree attention verifies all branches in one target forward pass using a sparse hierarchical mask where each token attends only to its ancestor nodes.

Example — four candidate completions of "machine ...":
```
machine → learning → algorithm → is
       → learning → system → design
       → translation → models → are
       → translation → system → design
```

Tree has 11 nodes. One target forward pass verifies all of them. Accept/reject by tree traversal using SpecJudge at each node — take the longest accepted path.

This is the mechanism behind EAGLE-2 and EAGLE-3's dynamic draft trees.

---

### Medusa — multiple draft heads inside one model

Add K extra LM heads to the final hidden layer of the target:
- Head 1 predicts token t+2
- Head 2 predicts token t+3
- ...
- Head K predicts token t+K+1

All heads produce candidates in parallel → combine into s₁×s₂×...×sK candidate sequences → verify via tree attention in one forward pass. Single model deployment; no separate draft weights.

|-------|------|

| Mixtral 8×7B | `text-generation-inference/Mixtral-8x7B-Instruct-v0.1-medusa` |
| Mistral 7B v0.2 | `text-generation-inference/Mistral-7B-Instruct-v0.2-medusa` |

## §G — Correction Distribution Code and Full Acceptance Scenarios

*Source: COMBINED.md §5.4 — from Brenndoerfer (08)*

*Omitted because:* The blog states the correction rule in prose. The code implementation belongs here for readers who want to implement it.*

```python
import numpy as np

def accept_probability(p_target: float, q_draft: float) -> float:
    return min(1.0, p_target / q_draft)

def correction_distribution(p_target: np.ndarray, q_draft: np.ndarray) -> np.ndarray:
    """
    Returns the residual distribution to sample from on rejection.
    Ensures the overall output distribution equals p_target exactly.
    """
    residual = np.maximum(0, p_target - q_draft)   # element-wise
    total = np.sum(residual)
    if total > 0:
        return residual / total      # normalized residual
    else:
        return p_target              # fallback: full target distribution
```

Full acceptance scenarios:

| Scenario | p_target | q_draft | P(accept) | Interpretation |
|----------|----------|---------|-----------|---------------|
| Target strongly prefers | 0.25 | 0.10 | 1.00 | Always accept — target is more confident than draft |
| Close agreement | 0.15 | 0.18 | 0.83 | Accept 83% of the time |
| Mild overestimate | 0.10 | 0.20 | 0.50 | Accept half the time |
| Strong overestimate | 0.05 | 0.20 | 0.25 | Accept 25% of the time |
| Extreme overestimate | 0.01 | 0.20 | 0.05 | Almost always reject |

The accepted samples + residual resamples combine to produce exactly `p_target` — not an approximation.

**Note on greedy decoding (temperature = 0):** The blog's mechanism is the general stochastic case. At temperature = 0, the acceptance criterion simplifies to: `accept if argmax(target_logits) == draft_token`. The correction on rejection simplifies to: always use the target's argmax. This is the special case Layer 14 implements.
